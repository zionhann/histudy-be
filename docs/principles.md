# 핵심 원칙

## 제품 방향

HIStudy는 학기 단위 스터디 그룹 운영을 예측 가능하게 만드는 서비스입니다.
학생은 원하는 과목과 함께하고 싶은 사람을 바탕으로 그룹에 신청하고, 운영자는 매칭과 학기 운영을 관리하며, 공개 사용자는 가입 없이 주요 활동 지표를 확인할 수 있어야 합니다.

## 운영 원칙

- 학사 학기는 서비스의 기본 운영 단위다.
- 활성 운영 문맥은 항상 하나의 현재 학기를 기준으로 잡는다.
- 매칭은 사회적 의도를 먼저, 과목 선호를 그다음 기준으로 삼는다.
- 공개 화면은 개인 식별 정보보다 집계된 결과를 우선한다.
- 관리자 기능은 숨겨진 자동화보다 명시적이고 이해 가능한 흐름을 유지한다.

## 동작 원칙

- 의도적으로 공개된 경로가 아니라면 보호된 API는 Bearer 인증을 요구한다.
- `USER`, `MEMBER`, `ADMIN`의 역할 경계는 명확하게 유지한다.
- 매칭은 두 단계로 처리한다.
  - 친구 우선 매칭: 최소 그룹 크기 2
  - 과목 우선 매칭: 목표 그룹 크기 3~5, 기준 미달 잔여 인원은 미배정 유지
- 그룹 식별자는 학기 안에서 고유한 태그로 유지한다.
- 그룹 과목은 멤버 선호의 겹침을 반영하고, 가능하면 2명 이상이 선택한 과목을 우선한다.
- 리포트 생성과 수정은 매칭된 멤버가 담당하며, 이미지 증빙과 경로 재사용을 지원한다.
- 사용자 대상 조회 API는 가능한 경우 마스킹된 식별자를 우선한다.

## 비목표와 경계

HIStudy는 다음 영역까지 확장하는 것을 목표로 하지 않는다.

- 실시간 채팅 또는 미팅 플랫폼
- LMS 연동 계층
- 결제 또는 보상 시스템
- 기관 맥락에서 분리된 범용 멀티테넌트 스터디 플랫폼

## 호환성 원칙

- deprecated 엔드포인트는 호환성을 위해 남아 있을 수 있다.
- 새로운 동작은 더 안전하고 최신인 표면을 우선하며, 특히 `v2` 마스킹 변형을 기준으로 삼는다.
- 공개 엔드포인트는 인증 없이 접근 가능해야 하며, 개인 식별 정보를 노출하지 않는다.

## 건강성 신호

- 학생이 미배정 상태에서 학기별 신청을 무리 없이 완료하고 다시 제출할 수 있다.
- 관리자가 수동 정리 없이 매칭을 실행하고 배정 상태를 확인할 수 있다.
- 매칭된 그룹이 리포트와 이미지 증빙을 꾸준히 남길 수 있다.
- 공개 활동 지표와 랭킹 화면이 프라이버시를 지키면서도 충분히 유용하다.
- 매칭 커버리지와 리포트 활동량이 안정적으로 유지되거나 개선된다.
